个人博客搭建入门教程

发表于 2019-01-26 | 分类于 博客搭建 | 阅读次数:
字数统计: 1.4k 字 | 阅读时长 ≈ 5 分钟

介绍

  • 本教程是采用Gitee或Github和Hexo框架来搭建个人博客的。
  • 本教程中的所有链接均指向为官方链接，望知悉。
  • 本教程中的部分内容是借鉴酷安教程所写，如需转载请注明。
  • 本教程中的所有软件的安装与调试都不赘述，望您多动手百度。

为什么用Gitee？

众所周知，Github在国内访问速度慢，还有可能被墙。而Gitee作为国内版的”Github”，速度快还不用担心后台可能会挂，也不用自己花钱买域名，所以用Gitee来构建我个人的第一个博客了。

为什么用Github？

大名鼎鼎的Github就不用介绍了吧，虽然国内访问速度慢，但可是大佬们的聚集地啊，想要进阶大佬，不得来一个Github的账号？

Hexo是什么？

官网直达车：查看链接

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。>

准备工作

  1. 一个Gitee账号。直达链接：查看链接
  2. 一个Github账号。直达链接：查看链接
  3. 安装Git（用来从本地传到Gitee端）。直达链接：查看链接
  4. 安装Node.js（用来本地编译生成静态网页）。直达链接：查看链接
  5. 安装文件查看器，用来修改本地的各种文件内容，本人使用的是VS code。不拘泥于一个文件，下方的Markdown编辑器也可以。直达链接：查看链接
  6. Atom（Github自家的Markdown编辑器，主要写博客时使用）。直达链接：查看链接
  7. 小书匠编辑器（Markdown编辑器，写博客时使用）。直达链接：查看链接

开始搭建

一.在Gitee中创建一个用来放博客的 仓库

1. 登入Gitee后先新建一个仓库

2. 下边选项默认即可

3. 开启Gitee Pages服务来托管你的博客

码云 Pages 是一个免费的静态网页托管服务，您可以使用 码云 Pages 托管博客、项目官网等静态网页。如果您使用过 Github Pages 那么您会很快上手使用码云的 Pages服务。目前码云 Pages 支持 Jekyll、Hugo、Hexo编译静态资源
源文档：查看链接

4. 在该页面内启用，显示如下：

该链接即为你的博客网站，请牢记该网址，后续会用到

5. 至此，仓库完成创建且已开启服务

二.在本地安装Hexo框架

1. 打开Node.js的命令行窗口

2. 用一键安装命令安装Hexo

命令为：npm install -g hexo-cli

3. 使用 hexo v 命令可以查看安装版本

4. 初始化Hexo，即在本地创建Hexo文件

命令为： hexo init "D:\My Hexo"
建议将该文件夹放在常用盘符下，后续操作均在该文件夹内操作

注意：如果一直卡在箭头这里，请移步：[解决办法]。之后再次进行初始化命令即可正常解决。

5. 进入你的数据目录

命令为： cd "D:\My Hexo"

6. 再次初始化安装Hexo

命令为： npm install

我这里已经装过了，可能界面显示内容和你们不一样，别急，先进行下一步。如果不对再返回来重试。

7. 打开 My Hexo 目录，发现有以下文件即为正确

8. 检测本地Hexo是否安装成功

先在上边目录右键进入Git的命令行模式

输入 hexo s 开启本地服务

复制上图选框内容，去浏览器打开

9. 至此，本地博客已安装成功。

三.配置上传服务

1. 找到码云的设置界面的这两处信息

2. 在Git界面配置Git全局信息

  • 命令为： git config --global user.name "你的昵称"
  • 命令为： git config --global user.email "你的邮箱"

3. 修改本地配置文件

在你的数据目录内找到这个文件 _config.yml 并打开

找到URL一行，修改url和root。url即为这里的博客地址，而绿字解释了为什么root这样写。

之后按ctrl+S保存

4. 设置Git路径

进入你创建的仓库，在右边克隆那复制HTTPS地址

打开 _config.yml 配置文件，修改最下边deploy区域，按图增加一个repo

注意：图中红线框着的地方有空格，千万不要忘记。这里是博主遇到的最大的坑。Hexo中和Markdown中最要注意的就是空格了。

之后按ctrl+S保存

5. 安装hexo-deployer-git插件

命令为： npm install hexo-deployer-git --save

6. 生成静态文件并上传至Gitee端，将博客部署到服务器

命令为： hexo g && hexo d
中途会让你输入Gitee的账户密码


这就代表上传完毕了

7. 进入仓库检查是否上传成功

这就代表已成功传入仓库中

8. 浏览器登录博客地址查看博客

9. 至此，博客已经搭建完毕

PS：常用指令

  • hexo clean 清理缓存文件——db.json和已生成的静态文件——public
  • hexo g 生成网站静态文件到默认设置的 public 文件夹。
  • hexo s 启动本地服务器，用于预览主题。默认地址：http://localhost:4000/
  • hexo d 自动生成网站静态文件，并部署到设定的仓库。
